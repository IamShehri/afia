# docs/

Product documentation and release evidence (changes together with code).

- `product/` — engineering program, task ledger, journeys, support matrix.
- `architecture/` — current diagrams and boundary descriptions.
- `adr/` — architecture decision records.
- `security/` — threat model, data-flow inventory, control verification.
- `quality/` — model cards, evaluation reports, benchmarks, OpenMed inventory/licenses.
- `runbooks/` — operational procedures exercised before release.
- `releases/` — immutable per-release evidence (e.g. `v1.0.0/`), created during the release phase.
